## 1. 模块概述

报表中心为加气站运营管理系统提供**结构化报表生成、查阅与导出**能力。与 7.1 数据分析（交互式可视化探索）互补，本模块侧重于**固定格式的定期报表**和**用户可配置的自定义报表**，满足日常运营汇报、财务审计和安全合规需求。

### 1.1 模块定位

| 维度 | 说明 |
|------|------|
| 用途 | 生成标准格式的经营/财务/安全报表；支持自定义维度指标组合 |
| 数据来源 | Phase 1-2 已有模块的业务数据（聚合查询），与 7.1 共享数据源 |
| 自有实体 | 报表模板（ReportTemplate）、报表实例（ReportInstance） |
| 写操作 | 创建/编辑报表模板、生成报表实例、标记收藏 |
| 导出格式 | Excel (.xlsx)、PDF（MVP 阶段 mock 导出按钮，实际文件生成留后端） |

### 1.2 与相关模块关系

| 模块 | 关系 |
|------|------|
| 7.1 数据分析 | 共享数据源（FuelingOrder、Station 等）；7.1 做交互式探索，7.2 做固定格式报表 |
| 7.3 数据大屏 | 报表中心可提供大屏所需的数据源 API |
| 5.1 财务管理（Phase 6） | ⚠️ 财务报表部分与 5.1 有交叉，本模块仅做运营视角的财务汇总 |
| 1.4 巡检安检管理 | 巡检报表的数据来源 |
| 1.2 交接班管理 | 交接班报表的数据来源 |

## 2. 功能需求

### Epic 1: 标准报表（Standard Reports）

预定义的固定格式报表，覆盖日常运营核心场景。

#### F-001 经营日报

- 按日生成经营汇总报表
- 包含：营业额、订单数、充装量、客单价、同比/环比
- 按站点分列数据
- 支持查看历史日报

#### F-002 经营月报

- 按月生成经营汇总报表
- 包含：月度营收汇总、日均营收趋势图、站点排名、燃料类型占比
- 支持与上月/去年同月对比

#### F-003 交接班报表

- 按班次生成交接班汇总
- 包含：班次营收、订单数、充装量、支付方式分布
- ⚠️ 数据来源：1.2 交接班管理模块

#### F-004 巡检报表

- 按月/周生成巡检执行汇总
- 包含：计划完成率、问题发现数、整改完成率、按区域分布
- ⚠️ 数据来源：1.4 巡检安检管理模块

#### F-005 库存报表

- 按日/周生成库存变动汇总
- 包含：入库量、出库量、期末库存、损耗量、罐容比
- ⚠️ 数据来源：2.3 库存管理模块

#### F-006 报表日历视图

- 以日历形式展示已生成的历史报表
- 点击日期查看当日可用报表
- 直观展示报表生成频率和覆盖情况

### Epic 2: 自定义报表（Custom Reports）

用户可配置维度和指标组合，创建个性化报表。

#### F-007 报表模板创建

- 用户选择数据源（经营/库存/巡检）
- 配置维度（时间粒度、站点、燃料类型等）
- 选择指标（营业额、订单数、充装量等）
- 设置筛选条件
- 保存为可复用的报表模板

#### F-008 报表模板管理

- 查看所有报表模板列表
- 编辑已有模板的配置
- 删除不需要的模板
- 模板标签分类管理

#### F-009 报表生成与预览

- 基于模板选择时间范围生成报表
- 支持表格和图表两种预览模式
- 预览满意后可导出

#### F-010 报表导出

- 导出为 Excel (.xlsx) 格式
- 导出为 PDF 格式
- MVP 阶段：展示导出按钮 + mock 下载提示，实际文件生成留后端实现

### Epic 3: 报表管理（Report Management）

#### F-011 报表收藏与快速访问

- 收藏常用报表，在首页快速访问区展示
- 最近查看的报表记录

#### F-012 报表列表与搜索

- 全量报表实例列表
- 按类型、日期范围、站点筛选
- 关键词搜索

## 3. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NF-01 | 响应性能 | 报表预览在 3 秒内加载完成（Mock 阶段即时） |
| NF-02 | 数据精度 | 金额精度 NUMERIC(12,2)；百分比保留 1 位小数 |
| NF-03 | 导出容量 | 单次导出最多 10,000 行数据（生产环境约束） |
| NF-04 | 权限控制 | 站长仅查看本站报表；运营经理可跨站查看 |

## 5. 约束与假设

### 约束

1. MVP 阶段使用 mock 数据，报表生成为即时模拟
2. 导出功能 MVP 阶段仅展示 UI，不生成实际文件
3. 报表数据精度遵循 STANDARDS.md 规范

### 假设

1. 用户已有 7.1 数据分析的使用经验，理解维度/指标概念
2. 标准报表模板由系统预置，用户不可修改其结构
3. 自定义报表模板仅创建者可见（MVP 阶段无共享机制）
