## 1. 功能拆解清单

### 1.1 订单查看与记录（来源：需求文档 §2.2 表格第1行）

**功能描述：** 按枪号、燃料、数量、金额、时间、车牌、会员等展示与查询订单记录。站端实时数据采集，管理后台支持智能搜索与多维度筛选。

| # | 功能点（L4 原子功能） | 描述 | 优先级 | 来源 |
|---|---------------------|------|--------|------|
| F001 | 订单列表 | 展示当前站点全部订单记录，显示订单号、枪号、燃料类型、数量(kg)、金额(元)、时间、支付状态 | [MVP] | §2.2 行1 |
| F002 | 订单筛选 | 按时间范围、枪号、燃料类型、支付状态、支付方式进行筛选 | [MVP] | §2.2 行1 |
| F003 | 订单搜索 | 按订单号、车牌号、会员手机号进行模糊搜索 | [MVP] | §2.2 行1 |
| F004 | 订单详情 | 查看单条订单的完整信息：订单基本信息、加注明细、支付明细、优惠明细、会员/车辆信息 | [MVP] | §2.2 行8 |
| F005 | 订单统计概览 | 页面顶部展示当日/当班统计卡片：总订单数、总金额、总充装量、各支付方式占比 | [MVP] | §2.2 行1 |

### 1.2 订单流（来源：需求文档 §2.2 表格第2行）

**功能描述：** 订单从加注开始到支付完成的全流程状态流转，包含订单号生成、状态管理、订单信息完善。

| # | 功能点（L4 原子功能） | 描述 | 优先级 | 来源 |
|---|---------------------|------|--------|------|
| F006 | 订单状态展示 | 实时展示订单状态：加注中(filling) → 待支付(pending_payment) → 已支付(paid) → 已完成(completed) | [MVP] | §2.2 行2 |
| F007 | 订单号生成 | 系统自动生成唯一订单号，格式：站点编码-日期-序号（如 ST001-20260225-0001），保证全局唯一 | [MVP] | §2.2 行2 |
| F008 | 手动创建订单 | 管理后台手动创建订单：选择枪号 → 系统自动获取当前价格 → 输入充装量 → 计算金额 → 生成订单。真实订单主要由站端 POS 通过设备数据采集生成，管理后台为补充手段 | [MVP] | §2.2 行2 |
| F009 | 订单取消 | 加注中或待支付状态的订单可取消，记录取消原因，订单状态变更为 cancelled | [MVP] | §2.2 行2 |
| F010 | 合并订单 | 同一车辆/会员的多笔充装合并为一笔订单结算 | [PROD] | §2.2 行2 |
| F011 | 拆单 | 一笔订单拆分为多笔子订单，分别结算 | [PROD] | §2.2 行2 |

### 1.3 支付处理（来源：需求文档 §2.2 表格第3-4行）

**功能描述：** 多种支付方式支持，含支付弹窗、金额计算、优惠应用、合并/拆分支付、小票打印。

| # | 功能点（L4 原子功能） | 描述 | 优先级 | 来源 |
|---|---------------------|------|--------|------|
| F012 | 支付方式选择 | 支持多种支付方式：现金、微信支付、支付宝、银联、会员卡、储值卡、IC卡、线下挂账、ETC | [MVP] | §2.2 行3 |
| F013 | 支付弹窗/面板 | 展示订单金额、可用优惠、应付金额，选择支付方式，确认支付 | [MVP] | §2.2 行4 |
| F014 | 金额计算 | 根据充装量 × 单价计算订单金额，应用优惠后计算实付金额 | [MVP] | §2.2 行4 |
| F015 | 混合支付 | 同一笔订单使用多种支付方式组合支付（如部分现金 + 部分微信） | [MVP+] | §2.2 行3 |
| F016 | 支付状态追踪 | 展示支付状态（待支付、支付中、支付成功、支付失败），支持支付超时倒计时 | [MVP] | §2.2 行4 |
| F017 | 支付确认回调 | 第三方支付成功后回调更新订单状态 | [MVP] | §2.2 行3 |
| F018 | 小票信息预览 | 展示小票预览：订单编号、枪号、燃料类型、单价、数量、金额、支付方式、时间、标签/备注 | [MVP+] | §2.2 行4 |

### 1.4 异常订单处理（来源：需求文档 §2.2 表格第5行）

**功能描述：** 掉电/断网等异常场景下的未付订单处理，包含订单拦截、挂起、补单与审核机制。

| # | 功能点（L4 原子功能） | 描述 | 优先级 | 来源 |
|---|---------------------|------|--------|------|
| F019 | 异常订单标记 | 系统自动识别异常订单（掉电未付、超时未付、金额异常），标记异常类型和原因 | [MVP] | §2.2 行5 |
| F020 | 异常订单列表 | 单独的异常订单视图，按异常类型、时间、处理状态筛选 | [MVP] | §2.2 行5 |
| F021 | 订单挂起 | 将异常订单挂起等待处理，挂起期间订单不参与班次结算 | [MVP] | §2.2 行5 |
| F022 | 补单操作 | 为异常订单补充支付信息，完成结算。记录补单原因、操作人、操作时间 | [MVP] | §2.2 行5 |
| F023 | 异常订单审核 | 补单操作需上级审核确认，审核通过后订单状态变更为已完成 | [MVP+] | §2.2 行5 |
| F024 | 异常原因记录 | 每笔异常订单记录异常原因（掉电、设备故障、客户离场等），支持自定义原因 | [MVP] | §2.2 行5 |

### 1.5 退款管理（来源：需求文档 §2.2 表格第6行）

**功能描述：** 退款申请、审批流程、退款执行、退款流水记录，确保操作可追溯。

| # | 功能点（L4 原子功能） | 描述 | 优先级 | 来源 |
|---|---------------------|------|--------|------|
| F025 | 退款申请 | 对已支付订单发起退款申请，填写退款原因、退款金额（支持全额/部分退款） | [MVP] | §2.2 行6 |
| F026 | 退款审批 | 退款申请提交后需审批，审批通过后执行退款。展示审批列表与审批操作 | [MVP] | §2.2 行6 |
| F027 | 退款执行与记录 | 审批通过后执行退款，记录退款流水（退款金额、退款方式、退款时间、操作人） | [MVP] | §2.2 行6 |
| F028 | 退款状态追踪 | 展示退款状态（申请中、审批中、退款中、已退款、已拒绝），退款进度可查询 | [MVP] | §2.2 行6 |

### 1.6 标签与备注（来源：需求文档 §2.2 表格第7行）

**功能描述：** 为订单添加标签（如"滴滴(6.05)"）和备注信息，便于分类统计和企业对账。

| # | 功能点（L4 原子功能） | 描述 | 优先级 | 来源 |
|---|---------------------|------|--------|------|
| F029 | 订单标签管理 | 为订单添加/移除标签，支持多标签。系统预设标签 + 自定义标签 | [MVP+] | §2.2 行7 |
| F030 | 订单备注 | 为订单添加文本备注信息 | [MVP] | §2.2 行7 |
| F031 | 标签配置 | 管理系统预设标签列表，支持新增、编辑、删除、排序 | [MVP+] | §2.2 行7 |
| F032 | 按标签筛选 | 在订单列表中按标签筛选订单，便于企业对账 | [MVP+] | §2.2 行7 |

**功能描述：** 按条件导出订单数据为 Excel/PDF，用于财务对账和报表生成。

| F035 | 导出为 PDF | 按条件导出订单汇总报表为 PDF 格式 | [PROD] | §2.2 行9 |

## 2. 依赖关系

### 2.1 模块内依赖

- F001（订单列表）→ F002, F003: 列表页面包含筛选和搜索功能
- F006（订单状态展示）→ F008, F009: 订单创建和取消触发状态变更
- F008（订单创建）→ F012, F013, F014: 创建订单后进入支付流程
- F012（支付方式选择）→ F016, F017: 选择支付方式后追踪支付状态
- F019（异常订单标记）→ F020, F021, F022: 标记异常后可查看、挂起、补单
- F025（退款申请）→ F026, F027, F028: 申请退款后进入审批和执行流程
- F033（订单导出）← F002: 导出使用列表筛选条件

### 2.2 跨模块依赖

| 功能点 | 依赖模块 | 依赖说明 |
|--------|----------|----------|
| F001, F008（订单列表 + 创建） | 站点管理 (1.1) | 依赖 Station、Nozzle、FuelType 实体，订单归属站点+枪+燃料类型 |

| F033, F035（订单导出） | — | 无外部依赖，纯前端导出功能 |

## 3. 疑问确认记录

**确认日期：** 待确认

| # | 问题 | 用户确认结果 |
|---|------|-------------|
| Q1 | 管理后台订单创建是模拟还是需要对接实际设备数据？ | **已确认**：真实订单主要由站端 POS 通过设备数据采集生成，管理后台也应支持手动创建订单（作为补充手段） |
| Q2 | 退款审批是否需要多级审批？ | **已确认**：MVP 阶段单级审批（站长/运营经理）。审批流程应设计为可配置的系统通用模块，标记待 Phase 7 (9.5) 实现自定义审批流 |
| Q3 | 异常订单补单是否扣减库存？ | **待确认**（建议：补单仅补录支付信息，充装量已在加注时扣减） |

| Q5 | 支付方式中哪些在 MVP 阶段实现？ | **已确认**：MVP 实现现金、微信、支付宝、银联 4 种常用方式，其余标记 [MVP+] |
| Q6 | 会员/车辆信息在订单中如何处理（无 Phase 4 模块）？ | **已确认**：与 2.1 价格管理同策略，UI 使用 mock 数据 + 标注"依赖会员/车辆模块" |

## 6. 参考的角色与职能

基于 `knowledge-base/LNG加气站核心人员职能及协作流程.md` 及需求文档 §六，订单与交易模块涉及的主要角色：

| 角色 | 与订单交易的关系 |
|------|-----------------|
| 收银员 | 日常订单处理的主要操作者，完成收银、支付确认、小票打印 |
| 加液操作员 | 执行充装操作，生成充装记录（数量、时间、枪号） |
| 站长 | 异常订单处理、退款审批、补单审核 |
| 运营经理 | 多站点订单数据查看、退款审批、异常订单监控 |
| 财务人员 | 订单数据导出、财务对账、退款流水核对 |
| 引导员 | 车辆引导、协助记录车牌信息 |
