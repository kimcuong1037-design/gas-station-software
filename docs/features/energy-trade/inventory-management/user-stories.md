## 角色说明

本模块涉及的用户角色：

| 角色 | 代码 | 说明 |
|------|------|------|
| 站长 | `station_master` | 站点库存管理、入库审核、盘点调整审批、预警配置 |
| 班组长 | `shift_leader` | 日常入库操作、损耗登记、罐存巡查、小额审批 |
| 运营经理 | `ops_manager` | 多站点库存监控、偏差分析、损耗追踪、预警总览 |
| 财务人员 | `finance` | 进销存流水核对、库存成本核算、数据导出 |

## Epic 1: 库存总览

来源功能点：F01, F02

### US-001: 查看库存总览仪表盘

**优先级：** [MVP]
**功能点：** F01
**角色：** 站长

> 作为站长，
> 我希望在一个仪表盘中查看当前站点各燃料类型的库存状态，
> 以便快速掌握整体库存水位和当日进出库变化。

**验收标准：**

1. 假设当前站点有库存数据，当进入库存总览页面，那么按燃料类型（LNG、CNG、L-CNG）分卡片展示：当前库存量(kg)、罐容比(%)、今日入库量、今日出库量、今日净变化。
2. 假设卡片中显示罐容比，当查看罐容比，那么以进度条形式呈现，颜色随安全/预警/紧急三级变化（绿→橙→红）。
3. 假设当前站点某燃料类型无库存数据，当查看仪表盘，那么对应卡片显示"暂无数据"。
4. 假设切换站点选择器，当选择不同站点，那么仪表盘数据刷新为目标站点的库存状态。

**关联：** US-002
**前置依赖：** 无

---

### US-002: 查看库存趋势图

**优先级：** [MVP]
**功能点：** F02
**角色：** 站长

> 作为站长，
> 我希望查看各燃料类型近期的库存变化趋势，
> 以便预判补货时机和分析库存消耗规律。

**验收标准：**

1. 假设当前站点有历史库存数据，当查看库存趋势图，那么展示折线图，X 轴为日期、Y 轴为库存量(kg)，按燃料类型分色展示。
2. 假设需要查看不同时间范围，当切换时间维度（近 7 天 / 近 30 天），那么图表数据和 X 轴范围相应更新。
3. 假设历史数据不足 7 天，当选择近 7 天，那么仅展示已有数据的日期范围。

**关联：** US-001
**前置依赖：** US-001

## Epic 2: 入库管理

来源功能点：F03, F04, F08, F09

### US-003: 查看入库记录

**优先级：** [MVP]
**功能点：** F03
**角色：** 站长

> 作为站长，
> 我希望查看当前站点的所有入库记录，
> 以便追溯每次卸液/进气的详细信息。

**验收标准：**

1. 假设当前站点有入库记录，当进入入库记录页面，那么显示入库列表：入库单号、供应商、送货单号、燃料类型、计划量(kg)、实收量(kg)、入库偏差、入库时间、操作员、审核状态。
2. 假设列表有多条记录，当查看列表，那么按入库时间倒序排列，支持分页浏览。
3. 假设需要按条件筛选，当使用时间范围、燃料类型、审核状态等筛选器，那么列表过滤显示匹配记录。
4. 假设需要查看入库详情，当点击某条记录，那么展开详情展示完整入库信息（含目标储罐、入库偏差计算、审核链）。

**关联：** US-004, US-005
**前置依赖：** 无

---

### US-004: 创建入库单

**优先级：** [MVP]
**功能点：** F04
**角色：** 班组长

> 作为班组长，
> 我希望在卸液/进气完成后录入入库信息，
> 以便更新理论库存并留存入库凭证。

**验收标准：**

1. 假设在入库记录页面，当点击"新增入库"按钮，那么打开入库单创建表单。
2. 假设填写入库信息，当选择目标储罐，那么系统自动关联该储罐对应的燃料类型，无需手动选择。
3. 假设输入计划量和实收量，当实收量 < 计划量，那么自动计算入库偏差并高亮显示（运输损耗提示）。
4. 假设入库信息填写完成（供应商、送货单号、计划量、实收量），当提交入库单，那么入库单状态为"待审核"，理论库存暂不更新（等待审核通过）。
5. 假设未填写必填字段（实收量、供应商），当提交，那么表单显示校验错误提示。

**关联：** US-003, US-005
**前置依赖：** 无

### US-005: 审核入库单

**优先级：** [MVP+]
**功能点：** F08
**角色：** 站长
**⚠️ 跨阶段依赖：** 审批流程引擎（Phase 7 模块 9.5）— MVP 阶段单级审批

> 作为站长，
> 我希望审核班组长提交的入库单，
> 以便确保入库数据准确并正式入账更新理论库存。

**验收标准：**

1. 假设有待审核的入库单，当进入审核列表，那么显示待审核入库单（入库单号、供应商、燃料类型、实收量、金额、提交人、提交时间）。
2. 假设审核通过，当点击"通过"，那么入库单状态变更为"已通过"，理论库存增加实收量。
3. 假设审核驳回，当点击"驳回"并填写原因，那么入库单状态变更为"已驳回"，理论库存不变，通知提交人。
4. 假设入库单金额 ≤ 5 万元，当提交审核，那么班组长可审批；金额 > 5 万元需站长审批。

**关联：** US-004
**前置依赖：** US-004

### US-006: 管理供应商

**优先级：** [MVP+]
**功能点：** F09
**角色：** 站长

> 作为站长，
> 我希望维护供应商列表，
> 以便入库时快速选择供应商，并保留供应商基础信息。

**验收标准：**

1. 假设在库存管理设置中，当进入供应商管理页面，那么显示供应商列表（名称、联系人、联系电话、地址、状态）。
2. 假设需要新增供应商，当点击"新增"并填写信息提交，那么供应商添加到列表中，入库时可在下拉列表中选择。
3. 假设需要编辑供应商信息，当修改并保存，那么供应商信息更新，已关联的历史入库记录保持原快照不变。
4. 假设需要停用供应商，当切换状态为"停用"，那么该供应商不再出现在入库单的下拉选项中，但历史记录中仍可查看。

**关联：** US-004
**前置依赖：** 无

## Epic 3: 出库管理

来源功能点：F05, F06, F22

### US-007: 查看出库记录

**优先级：** [MVP]
**功能点：** F05, F22
**角色：** 站长

> 作为站长，
> 我希望查看当前站点的所有出库记录（含销售自动出库和损耗出库），
> 以便全面掌握燃料流出情况。

**验收标准：**

1. 假设当前站点有出库记录，当进入出库记录页面，那么显示出库列表：出库单号、出库类型（销售/损耗）、燃料类型、数量(kg)、关联订单号、出库时间、操作来源（自动/手动）。
2. 假设订单完成（paid/completed）后自动生成销售出库记录，当查看出库列表，那么销售出库记录标记为"自动"，可点击关联订单号跳转订单详情。
3. 假设需要按条件筛选，当使用出库类型、时间范围、燃料类型筛选器，那么列表过滤显示匹配记录。
4. 假设订单退款（refunded）后系统自动生成冲红入库记录，当查看流水，那么冲红记录以红色标识并关联原出库单。

**关联：** US-008, US-009
**前置依赖：** 无

### US-008: 登记损耗出库

**优先级：** [MVP]
**功能点：** F06
**角色：** 班组长

> 作为班组长，
> 我希望登记非销售的燃料出库（蒸发损耗、泄漏、站间调拨等），
> 以便确保理论库存与实际情况一致。

**验收标准：**

1. 假设在出库记录页面，当点击"登记损耗出库"按钮，那么打开损耗出库登记表单。
2. 假设填写损耗信息，当选择燃料类型并输入损耗量(kg)，那么系统按当前价格自动计算损耗金额。
3. 假设选择损耗原因（蒸发损耗、泄漏、调拨、其他），当选择"其他"，那么需手动输入原因说明。
4. 假设损耗出库提交，当提交成功，那么损耗记录状态为"待审批"（均需班组长审批），理论库存暂不扣减。
5. 假设损耗审批通过，当审批完成，那么理论库存扣减损耗量。

**关联：** US-007
**前置依赖：** 无

## Epic 5: 罐存比对

来源功能点：F10, F11, F12, F13, F14, F15, F23, F24

### US-010: 查看罐存实时面板

**优先级：** [MVP]
**功能点：** F10, F11, F23, F24
**角色：** 站长

> 作为站长，
> 我希望在一个面板中查看每个储罐的实时液位和理论库存对比，
> 以便及时发现罐存异常。

**验收标准：**

1. 假设当前站点有储罐设备，当进入罐存比对页面，那么以卡片形式展示每个储罐：储罐名称、燃料类型、额定容量、实际罐存(kg)、理论库存(kg)、偏差值、偏差率(%)。
2. 假设实际罐存数据来自设备监控（EquipmentMonitoring），当液位传感器更新数据，那么面板数据同步刷新（间隔 ≤ 15 秒）。
3. 假设理论库存按公式计算（期初 + Σ入库 - Σ销售出库 - Σ损耗出库 ± Σ盘点调整），当查看面板，那么理论库存值实时反映最新进出库情况。
4. 假设偏差率超过配置阈值（默认 ±2%），当查看面板，那么对应储罐卡片偏差值标红显示。

**关联：** US-011, US-012
**前置依赖：** 无

---

### US-011: 查看偏差分析

**优先级：** [MVP]
**功能点：** F12
**角色：** 站长

> 作为站长，
> 我希望查看详细的偏差分析数据，
> 以便判断偏差是否在合理范围内或需要采取措施。

**验收标准：**

1. 假设罐存面板显示偏差数据，当点击某储罐的"偏差分析"，那么展示偏差详情：偏差值 = 实际罐存 - 理论库存、偏差率 = 偏差值 / 理论库存 × 100%。
2. 假设偏差值为正（实际 > 理论），当查看分析，那么标注为"盈余"（绿色）。
3. 假设偏差值为负（实际 < 理论），当查看分析，那么标注为"亏损"（红色），并给出可能原因提示（泄漏、计量误差、未登记损耗）。
4. 假设偏差率超过阈值，当查看分析，那么显示预警提示并建议执行盘点调整。

**关联：** US-010, US-014
**前置依赖：** US-010

### US-013: 查看损耗分类分析

**优先级：** [MVP]
**功能点：** F14
**角色：** 运营经理

> 作为运营经理，
> 我希望查看运输损耗和站内损耗的分类统计，

**验收标准：**

1. 假设有入库偏差和日常偏差数据，当进入损耗分类分析页面，那么分两栏展示：运输损耗（入库偏差汇总）、站内损耗（日常偏差汇总）。

3. 假设查看站内损耗，当查看统计，那么展示各储罐的日均偏差率和损耗趋势。
4. 假设损耗超过行业基准值，当查看分析，那么以警示色标注异常项。

**关联：** US-012

### US-014: 执行盘点调整

**优先级：** [MVP+]
**功能点：** F15
**角色：** 站长
**⚠️ 跨阶段依赖：** 审批流程引擎（Phase 7 模块 9.5）— MVP 阶段单级审批

> 作为站长，
> 我希望在偏差超限时通过盘点调整使理论库存与实际罐存一致，
> 以便消除累积偏差，恢复库存数据准确性。

**验收标准：**

1. 假设罐存偏差超过阈值，当在偏差分析页面点击"盘点调整"，那么打开盘点调整表单，预填当前理论库存和实际罐存。
2. 假设填写调整信息（调整量、调整原因），当提交盘点调整，那么调整单进入审批流程（调整量 ≤ 理论库存 1% → 班组长审批，> 1% → 站长审批）。
3. 假设审批通过，当盘点调整生效，那么理论库存按调整量更新，并生成一条"盘点调整"类型的进销存流水。
4. 假设审批驳回，当驳回盘点调整，那么理论库存不变，记录驳回原因，通知发起人。

**关联：** US-011
**前置依赖：** US-010

## Epic 6: 库存预警

来源功能点：F16, F17, F18, F19, F20, F21

### US-015: 接收低库存预警

**优先级：** [MVP]
**功能点：** F16
**角色：** 站长

> 作为站长，
> 我希望在库存低于预设阈值时收到预警通知，
> 以便及时安排补货，避免断供。

**验收标准：**

1. 假设某燃料类型罐容比 ≤ 30%（预警阈值），当系统检测到低库存，那么生成"预警"级别通知（橙色），提示"建议补货"。
2. 假设某燃料类型罐容比 ≤ 10%（紧急阈值），当系统检测到极低库存，那么生成"紧急"级别通知（红色），提示"立即补货"。
3. 假设库存回升至安全阈值以上，当系统检测到库存恢复，那么预警自动消除，状态变为"已恢复"。

**关联：** US-016, US-017
**前置依赖：** 无

---

### US-016: 配置预警阈值

**优先级：** [MVP]
**功能点：** F17, F20
**角色：** 站长

> 作为站长，
> 我希望按站点和燃料类型配置多级预警阈值和通知规则，
> 以便根据实际运营情况调整预警灵敏度。

**验收标准：**

1. 假设进入预警规则配置页面，当查看配置项，那么按燃料类型分行展示三级阈值：安全(绿)、预警(橙)、紧急(红)，每级可配置百分比(罐容比)或绝对值(kg)。
2. 假设修改 LNG 的预警阈值（如 30% → 25%），当保存配置，那么新阈值立即生效，后续按新阈值判定预警。
3. 假设需要配置损耗异常阈值，当设置日偏差率阈值（默认 ±2%），那么超出此阈值时自动触发损耗异常预警。
4. 假设需要配置通知方式，当设置通知对象，那么可选择页面通知（Demo 阶段仅支持此方式）。

**关联：** US-015, US-017
**前置依赖：** 无

---

### US-017: 查看预警通知列表

**优先级：** [MVP]
**功能点：** F18
**角色：** 站长

> 作为站长，
> 我希望查看当前活跃预警和历史预警记录，
> 以便了解预警处理情况。

**验收标准：**

1. 假设有活跃预警，当进入预警通知页面，那么分为"活跃预警"和"历史预警"两个标签页，活跃预警优先展示。
2. 假设查看活跃预警列表，当查看记录，那么显示：预警级别（橙/红）、预警类型（低库存/损耗异常）、燃料类型、当前值、阈值、触发时间。
3. 假设需要处理预警，当点击"确认"按钮，那么预警标记为"已确认"；点击"忽略"则标记为"已忽略"。
4. 假设查看历史预警，当查看记录，那么显示预警的完整生命周期：触发时间、确认时间、恢复/关闭时间、处理人。

**关联：** US-015, US-016
**前置依赖：** US-015

### US-018: 接收损耗异常预警

**优先级：** [MVP]
**功能点：** F19
**角色：** 运营经理

> 作为运营经理，
> 我希望在日偏差率超过阈值时收到损耗异常预警，
> 以便及时排查疑似泄漏或计量异常。

**验收标准：**

1. 假设某储罐日偏差率超过 ±2%（默认阈值），当系统每日比对完成后，那么自动生成损耗异常预警。
2. 假设生成损耗异常预警，当查看预警详情，那么显示：储罐名称、偏差值、偏差率、可能原因分析（泄漏、计量异常、未登记损耗）。
3. 假设连续多日出现损耗异常，当查看预警列表，那么预警升级为"紧急"级别并标注连续天数。

**关联：** US-015, US-017
**前置依赖：** US-015

### US-019: 查看预警仪表盘

**优先级：** [MVP+]
**功能点：** F21
**角色：** 运营经理

> 作为运营经理，
> 我希望在一个仪表盘中总览所有预警状态，
> 以便快速评估各站点的库存风险等级。

**验收标准：**

1. 假设有多条预警记录，当进入预警仪表盘，那么展示预警汇总：按级别分类的活跃预警数量、预警处理率统计（已确认/总数）。
2. 假设查看预警趋势，当查看仪表盘，那么展示近 7/30 天的预警数量趋势图（按级别分色）。
3. 假设某燃料类型预警频繁触发，当查看统计，那么该类型在排名中靠前显示，便于运营重点关注。

**关联：** US-017
**前置依赖：** US-017
