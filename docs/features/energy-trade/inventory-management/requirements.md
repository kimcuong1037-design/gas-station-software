# Module 2.3 库存管理 — 需求文档

**模块：** 能源交易 / 库存管理 (Inventory Management)
**版本：** 1.0
**创建日期：** 2026-02-28
**依赖模块：** 1.1 站点管理、1.3 设备设施管理、2.2 订单与交易

---

## 1. 模块概述

库存管理模块负责加气站燃料（LNG、CNG、L-CNG）的进销存全流程管理，包括入库记录、销售出库自动关联、罐存实时监控与比对分析、库存预警等。本模块是能源交易链条的末端环节，与订单交易模块和设备设施模块深度集成。

**核心价值：**
- 实时掌握各站点各燃料类型库存水位
- 理论库存与实际罐存自动比对，识别异常损耗
- 可配置的多级预警机制，防止断供
- 完整的进销存流水审计链

## 3. 业务规则

### 3.1 库存计算规则

| 规则 | 说明 |
|------|------|
| 理论库存 | `期初 + Σ入库 - Σ销售出库 - Σ损耗出库 ± Σ盘点调整` |
| 实际罐存 | 来自 EquipmentMonitoring.level_volume（传感器实测） |
| 偏差值 | `实际罐存 - 理论库存`（正值=盈余，负值=亏损） |
| 偏差率 | `偏差值 / 理论库存 × 100%` |
| 罐容比 | `实际罐存 / 储罐额定容量 × 100%` |
| 入库偏差 | `实收量 - 计划量`（运输损耗指标） |

### 3.2 出库自动化规则

- 订单状态变为 `paid` 或 `completed` 时，系统自动生成出库记录
- 出库记录关联 `orderId`、`fuelTypeId`、`quantity`、`nozzleId`
- 退款（`refunded`）时，系统自动生成冲红入库记录

### 3.3 预警触发规则

| 级别 | 默认阈值 | 触发条件 |
|------|---------|---------|
| 安全 (safe) | > 30% 罐容比 | 正常运营 |
| 预警 (warning) | ≤ 30% 罐容比 | 橙色提醒，建议补货 |
| 紧急 (critical) | ≤ 10% 罐容比 | 红色告警，立即补货 |
| 损耗异常 | 日偏差率 > ±2% | 疑似泄漏或计量异常 |

### 3.4 审批规则

- 入库单：金额 ≤ 5 万元 → 班组长审批；> 5 万元 → 站长审批
- 盘点调整：调整量 ≤ 理论库存的 1% → 班组长审批；> 1% → 站长审批
- 损耗出库：均需班组长审批

## 4. 非功能需求

| 项目 | 要求 |
|------|------|
| 数据精度 | 量：NUMERIC(10,3)（精确到 0.001 kg/m³）；金额：NUMERIC(12,2) |
| 时间精度 | 所有时间戳精确到秒，带时区 |
| 审计追踪 | 所有进出库记录不可修改（只追加），盘点调整需记录原因和审批链 |
| 实时性 | 罐存面板数据刷新间隔 ≤ 15 秒（与设备监控同步） |
| 历史保留 | 进出库明细永久保留；罐存比对快照保留 2 年 |

## 6. 约束与假设

- 本阶段为前端 UI 原型，库存数据使用 Mock 数据模拟
- 实际罐存来自设备监控模块的液位数据（Mock 中直接提供）
- 订单出库联动在 Mock 环境中通过关联数据实现，无需实时事件
- 供应商管理为简化版（MVP+ 阶段），不含资质证照上传
- 预警通知方式在 Demo 阶段仅支持页面内显示，不含短信/邮件
