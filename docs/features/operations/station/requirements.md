## 1. 功能拆解清单

### 1.1 站点基础信息（来源：需求文档 §1.1 表格第1行）

**功能描述：** 管理站点的核心属性，包括站点名称、编码、地址（经纬度）、联系方式、营业时间。支持多站点统一管理（连锁模式）、站点分组/区域管理。

| F002 | 新增站点 | 填写站点名称、编码、地址（含经纬度）、联系方式、营业时间，保存新站点 | [MVP] | §1.1 行1 |
| F003 | 编辑站点信息 | 修改已有站点的基础属性信息 | [MVP] | §1.1 行1 |
| F004 | 查看站点详情 | 查看单个站点的完整信息详情页 | [MVP] | §1.1 行1 |

| F006 | 多站点切换 | 在系统中切换当前操作的站点上下文 | [MVP] | §1.1 行1 说明 |

## 2. 依赖关系

### 2.1 模块内依赖

- F002（新增站点）→ F003, F004, F005: 需先有站点才能编辑、查看详情、删除
- F002（新增站点）→ F009-F016: 枪配置依赖站点存在
- F002（新增站点）→ F021-F024: 站点照片依赖站点存在
- F002（新增站点）→ F025-F033: 班次与排班依赖站点存在
- F026（新增班次定义）→ F029, F030: 排班计划依赖班次定义
- F032（员工列表）→ F030: 创建排班计划需要员工数据

### 2.2 跨模块依赖

| 功能点 | 依赖模块 | 依赖说明 |
|--------|----------|----------|
| F012（设置枪单价） | 能源交易 > 价格管理 (2.1) | 枪单价与价格管理模块存在数据同步或联动关系 |
| F013（查看枪实时状态） | 设备设施管理 (1.3) | 枪状态数据来源于设备监控模块的数据采集 |
| F016（加注机数据采集配置） | 设备设施管理 (1.3) | 与设备联网对接功能存在重叠，需明确边界 |
| F017-F020（充电桩相关） | 非油增值 > 充电系统 (6.4) | 充电桩映射与充电系统模块强依赖 |
| F032（员工列表） | 系统与权限 > 用户管理 (9.1) | 员工数据可能来源于系统用户管理模块 |
| F033（员工考勤管理） | 系统与权限 (9) | 考勤可能与系统权限、用户管理有关联 |
| F034-F037（责任/维护站点） | 会员与大客户 > 大客户管理 (3.2) | 责任站点/维护站点概念在大客户管理中使用 |

## 3. 疑问确认记录

**确认日期：** 2026-02-15

| # | 问题 | 用户确认结果 |
|---|------|-------------|
| Q1 | 站点编码生成规则？ | **支持两种方式**：系统自动生成 + 用户手工输入 |
| Q2 | 站点分组与区域管理的关系？ | **两个独立维度**：分组用于业务分类，区域用于地理层级 |
| Q3 | 删除站点时关联数据处理？ | **软删除**：标记为停用，保留历史数据 |
| Q4 | 员工数据维护方式？ | **可配置**：系统设置中提供开关，默认从系统用户模块同步；关闭则站点模块独立维护 |
| Q5 | 枪单价与价格管理模块关系？ | **站点模块设置基准价**，价格管理模块负责调价 |
| Q6 | 燃料类型是否支持自定义？ | **优先固定枚举**（92#、95#、98#、柴油、CNG、LNG），不在枚举内的支持自定义 |
| Q7 | 充电桩映射功能如何处理？ | **预留数据结构**，界面暂不展示 |
| Q8 | 考勤管理的范围？ | **简单出勤记录**（[PROD]阶段） |
| Q9 | 站点照片要求？ | 待细化（MVP 阶段使用通用图片上传） |

## 6. 参考的角色与职能

基于 `knowledge-base/LNG加气站核心人员职能及协作流程.md`，站点管理模块涉及的主要角色：

| 角色 | 与站点管理的关系 |
|------|-----------------|
| 站长 | 站点信息的主要维护者，负责站点全面运营管理 |
| 技术负责人/生产经理 | 可能参与设备配置（枪配置）相关操作 |
| 财务人员/统计员 | 使用站点维度进行数据统计与报表 |
| 运营人员（后台） | 管理多站点信息、分组、区域配置 |
