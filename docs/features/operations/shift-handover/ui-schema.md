## 1.5 全局 — 用户身份展示

**位置：** 页面右上角（Header 区域）
**对应 Story：** US-020

**展示内容：**

- 用户姓名（如"张建国"）
- 角色标签（如"站长"）

## 1.6 全局 — 站点选择器

**位置：** 交接班模块页面顶部（Header 内，面包屑右侧）
**对应 Story：** US-021

**展示内容：**
- 当前站点名称 + 下拉切换器（Select 组件）
- 站点列表来源：mock 数据
- 选择站点后：整个交接班模块的数据上下文切换为该站点

**交互行为：**
| 触发 | 行为 |
|------|------|
| 选择站点 | 更新当前站点上下文，刷新所有数据 |

## 3. 页面详细设计

### P01: 站点经营概况页 (StationOverview / ShiftSummary)

**路由：** `/operations/shift-handover`
**对应 Story：** US-023, US-001, US-002, US-003
**权限：** `station_master`, `shift_leader`, `cashier`

#### 页面头部

- **面包屑：** 首页 / 基础运营 / 交接班管理
- **标题：** 站点概况
- **站点选择器：** 下拉选择当前站点（如"北京朝阳加气站"），可切换
- **操作按钮：**
  - `[刷新]` (Default) → 手动刷新数据
  - `[👁️/🔒]` (Icon) → 切换金额脱敏

#### 当前班次信息栏 + 下一班次信息

```
┌─────────────────────────────────────────────────────────────────────────┐
│  当前班次                                                               │

└─────────────────────────────────────────────────────────────────────────┘
```

#### 核心指标区 (StatCards)

**布局：** 3列等宽卡片（响应式调整）

| 卡片 | 字段 | 图标 | 颜色 | 说明 |
|------|------|------|------|------|
| 营业额 | `totalAmount` | MoneyCollectOutlined | brand | 主数字 + 较上班次变化 |
| 交易笔数 | `totalOrders` | ShoppingCartOutlined | info | 主数字 + 较上班次变化 |
| 净充装量 | `fuelSummary` | FireOutlined | success | 按燃料类型分行显示 |

**卡片结构：**
```
┌──────────────────────────────────┐
│  💰 营业额                       │

│     ¥ 13,373.00                  │
```

**脱敏模式：**
- 金额显示为 `*****`
- 较上班次变化隐藏
- 脱敏状态保存到 localStorage

```

```

#### 交互行为

| 触发 | 行为 | 说明 |
|------|------|------|
| 点击 [刷新] | 立即刷新 + 加载动画 | 更新最后刷新时间 |
| 点击 [👁️/🔒] | 切换脱敏状态 | 金额显示/隐藏 |
| 点击 [交接班] | 跳转 P02 | 仅班组长/站长可用 |
| 点击 [现金解缴] | 跳转 P03 | 仅班组长/站长可用 |
| 自动刷新 | 每20秒自动刷新 | 页面可见时生效 |
| 快捷键 R | 手动刷新 | 全局 |
| 快捷键 H | 切换脱敏 | 全局 |

#### 响应式适配

| 断点 | 布局调整 |
|------|---------|
| ≥1200px | 核心指标3列，明细2列并排 |
| 768-1199px | 核心指标3列，明细堆叠 |
| <768px | 核心指标堆叠，明细堆叠，交接班按钮底部固定 |

### P02: 交接班向导 (ShiftHandoverWizard)

**路由：** `/operations/shift-handover/handover`
**对应 Story：** US-004, US-005, US-006, US-007, US-008, US-009
**权限：** `station_master`, `shift_leader`

#### 页面头部

- **面包屑：** 首页 / 基础运营 / 交接班管理 / 交接班
- **标题：** 交接班
- **操作按钮：**
  - `[取消]` (Default) → 返回班次汇总页

#### 步骤条 (Stepper)

**组件：** Fluent UI Stepper

| 步骤 | 标题 | 必填 | 说明 |
|------|------|------|------|
| 1 | 预检确认 | ✅ | 检查待处理事项 |
| 2 | 现金解缴 | ⚠️ | 有现金时必填 |
| 3 | 交接备注 | ❌ | 可选 |
| 4 | 确认交接 | ✅ | 选择接班人，完成交接 |

#### 步骤1: 预检确认 (PrecheckStep)

**组件：** Card + List

```
┌──────────────────────────────────────────────────────────────────────┐
│  交接班预检                                                          │
├──────────────────────────────────────────────────────────────────────┤
│  以下项目需要在交接班前确认：                                         │
│                                                                      │
│  ┌────────────────────────────────────────────────────────────────┐ │
│  │ ✅ 未完成订单       0 笔                             [已通过]  │ │
│  ├────────────────────────────────────────────────────────────────┤ │
│  │ ⚠️ 未解缴现金       ¥3,500.00                 [去解缴 →]      │ │
│  ├────────────────────────────────────────────────────────────────┤ │
│  │ ✅ 现金差异         ¥0.00                              [已通过]  │ │
│  ├────────────────────────────────────────────────────────────────┤ │
│  │ ✅ 异常订单         0 笔                               [已通过]  │ │
│  └────────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  💡 建议先完成现金解缴再继续交接班                                   │
│                                                                      │
│  [标注异常]                                                          │
├──────────────────────────────────────────────────────────────────────┤
│                                                [取消]  [下一步 →]   │
└──────────────────────────────────────────────────────────────────────┘
```

**预检项状态：**
| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| 通过 | CheckCircle | success | 无待处理项 |
| 警告 | Warning | warning | 有待处理项，可继续 |
| 失败 | XCircle | error | 必须处理才能继续 |

**交互行为：**
| 触发 | 行为 |
|------|------|
| 点击 [去解缴] | 跳转到步骤2 |
| 点击 [标注异常] | 打开异常标注弹窗 P07 |
| 点击 [下一步] | 进入步骤2 |
| 有失败项时 | 下一步按钮禁用，提示原因 |

#### 步骤2: 现金解缴 (CashSettlementStep)

**组件：** Form

| 字段 | 组件 | 必填 | 验证 | 说明 |
|------|------|------|------|------|
| 应收现金 | InputNumber (只读) | - | - | 系统计算，不可编辑 |
| 实收现金 | InputNumber | ✅ | > 0 | 用户输入 |
| 差额 | Display (自动计算) | - | - | 颜色标识长/短款 |
| 差异原因 | Select | 超阈值必填 | - | 差额 > ±50元时必填 |
| 备注说明 | TextArea | 选"其他"必填 | max 500 | 详细说明 |
| 解缴方式 | Radio.Group | ✅ | - | safe/bank/manager |
| 凭证照片 | Upload | ❌ | max 5张 | 可选 |

**差额显示规则：**
| 差额 | 类型 | 颜色 | 图标 |
|------|------|------|------|
| > 0 | 长款 | success (绿) | ArrowUp |
| < 0 | 短款 | error (红) | ArrowDown |
| = 0 | 平账 | neutral (灰) | Check |
