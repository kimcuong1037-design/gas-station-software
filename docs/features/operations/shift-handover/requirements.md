## 1. 模块概述

### 1.1 业务目标

交接班管理是加气站日常运营的核心环节，确保：
- 班次之间经营数据的准确交接
- 现金收入的及时解缴与核对
- 异常情况的及时发现与处理
- 完整的审计追踪与报表支持

### 1.2 模块边界

| 范围 | 说明 |
|------|------|
| **包含** | 班次汇总、交接班操作、现金解缴、交接班报表 |
| **不包含** | 班次定义（属于站点管理模块）、员工排班（属于站点管理模块） |

### 1.3 与其他模块关系

| 关联模块 | 关系说明 |
|----------|----------|
| 站点管理 (1.1) | 依赖班次定义、员工数据 |
| 订单与交易 (2.2) | 读取班次内的订单交易数据 |
| 财务管理 (5.1) | 现金解缴数据同步至财务模块 |
| 价格管理 (2.1) | 交接班时可能涉及价格调整 |

## 3. 功能点详情

### F001: 查看当前班次汇总

**功能描述：** 实时展示当前班次的经营数据汇总。

**输入：** 自动基于当前登录用户的站点和班次

**输出：**
- 营业额总计
- 交易笔数
- 充装量（按燃料类型细分）
- 按支付方式的金额分布

**业务规则：**
1. 数据实时从订单系统聚合计算
2. 仅显示当前班次开始后的数据
3. 默认20秒自动刷新一次

**UI 参考：**
- 卡片式仪表盘布局
- 支持数据脱敏开关

---

### F005: 发起交接班

**功能描述：** 当前班次负责人发起交接班流程。

**输入：**
- 交接备注（可选）
- 下一班次选择
- 接班人选择

**输出：**
- 交接班记录创建
- 通知接班人

**业务规则：**
1. 仅班次负责人或有权限的员工可发起
2. 发起前需完成交接班预检（F006）
3. 存在未处理异常时需确认或强制交接

**前置条件：**
- 当前班次处于进行中状态
- 用户具有交接班权限

---

### F011: 登记现金解缴

**功能描述：** 录入本班次的现金解缴信息。

**输入：**
- 实收现金金额
- 解缴金额
- 解缴方式（如：存入保险柜、银行存款、交给站长）
- 解缴凭证（可选）
- 备注

**输出：**
- 现金解缴记录
- 长短款计算结果

**业务规则：**
1. 系统应收现金 = 班次内现金支付订单总额
2. 长短款 = 实收现金 - 系统应收现金
3. 长短款超过阈值（如±50元）需填写说明

## 5. 数据字典

### 交接班状态

| 状态 | 说明 |
|------|------|
| `initiated` | 已发起，等待接班 |
| `pending_review` | 待审核（如有异常需审核） |
| `completed` | 已完成 |
| `cancelled` | 已取消 |

### 支付方式

| 代码 | 名称 | 分类 |
|------|------|------|
| `cash` | 现金 | 现金类 |
| `wechat` | 微信支付 | 电子支付 |
| `alipay` | 支付宝 | 电子支付 |
| `unionpay` | 银联 | 电子支付 |
| `ic_card` | IC卡 | 储值卡 |
| `member_card` | 会员卡 | 储值卡 |
| `credit` | 挂账 | 赊销 |
| `other` | 其他 | 其他 |

### 长短款类型

| 类型 | 说明 |
|------|------|
| `surplus` | 长款（实收 > 应收） |
| `shortage` | 短款（实收 < 应收） |
| `balanced` | 平账（差额在允许范围内） |

## 6. 非功能性需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 班次汇总查询响应时间 | < 2秒 |
| 交接班操作响应时间 | < 3秒 |
| 报表导出（1000条记录） | < 10秒 |

### 6.2 安全要求

| 要求 | 说明 |
|------|------|
| 操作审计 | 所有交接班操作需记录操作人、时间、IP |
| 敏感数据保护 | 金额数据支持脱敏显示 |
| 权限控制 | 强制交接班需高级权限 |

### 6.3 可用性要求

| 要求 | 说明 |
|------|------|
| 移动端适配 | 支持平板端操作 |
| 离线支持 | 网络中断时允许暂存本地，恢复后同步 |
| 打印支持 | 支持热敏小票打印和A4报表打印 |
