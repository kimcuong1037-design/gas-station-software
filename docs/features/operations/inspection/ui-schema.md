## 3. 页面详细设计

### P01: 安检任务列表页 (InspectionTaskList)

**路由：** `/operations/inspection/tasks`
**对应 Story：** US-006, US-009-A
**权限：** `safety_officer`, `station_master`, `inspector`, `ops_manager`

#### 页面头部

- **面包屑：** 首页 / 基础运营 / 巡检/安检管理
- **标题：** 安检任务
- **右侧信息：** 站点选择器（全局）

#### 顶部统计卡片区 (StatCards)

**布局：** 5列等宽卡片（Row + Col span≈5）

| 卡片 | 字段 | 图标 | 颜色 | 说明 | 点击行为 |
|------|------|------|------|------|---------|
| 今日任务 | `todayTotal` | CalendarOutlined | `#1890ff` | 当日任务总数 | 重置筛选到全部 |
| 已完成 | `todayCompleted` | CheckCircleOutlined | `#52c41a` | 当日已完成数 | 筛选状态=已完成 |
| 完成率 | `completionRate` | PieChartOutlined | `#1890ff` | 百分比 | - |
| 异常项 | `abnormalCount` | WarningOutlined | `#ff4d4f` | 当日异常检查项数，0时灰色 | 跳转巡检日志筛选结果=异常 |
| 待处理问题 | `pendingIssueCount` | BugOutlined | `#faad14` | 待处理问题总数 | 跳转问题记录列表 `/operations/inspection/issues` |

**卡片结构：**
```
┌──────────────────────────────────┐
│  📅 今日任务                     │
│                                  │
│        8                         │
│     (较昨日 +2)                  │
└──────────────────────────────────┘
```

#### 状态Tab切换

**组件：** Tabs

| Tab | 值 | Badge |
|-----|-----|-------|
| 全部 | `all` | 总数 |
| 待执行 | `pending` | 数量Badge |
| 执行中 | `in_progress` | 数量Badge |
| 已完成 | `completed` | - |

切换Tab = 设置任务状态筛选条件，刷新列表。

#### 筛选区域

| 字段 | 组件 | 数据源 | 默认值 | 宽度 |
|------|------|--------|--------|------|
| 关键词 | Input.Search | - | - | 240px |
| 执行人 | Select(带搜索) | 员工列表 API | 全部 | 150px |
| 截止日期 | DatePicker.RangePicker | - | 今日 | 280px |

**搜索框：** placeholder="搜索任务编号..."

#### 数据表格

| 列名 | 字段 | 宽度 | 对齐 | 渲染 | 排序 |
|------|------|------|------|------|------|
| 任务编号 | `taskNo` | 180px | left | 链接→P02，加粗 | - |
| 计划编号 | `plan.planNo` | 150px | left | 链接→P05，如 IP-HZXH-202602-001 | - |
| 关联计划 | `plan.name` | 160px | left | 链接→P05 | - |
| 执行人 | `assignee.name` | 100px | left | 文本，未分配时灰色"未分配" | - |
| 检查项 | `progress` | 80px | center | "已检/总数"格式（如 3/8） | - |
| 状态 | `status` | 100px | center | StatusTag | ✅ |
| 截止日期 | `dueDate` | 110px | center | 日期 YYYY-MM-DD | ✅ |
| 操作 | - | 120px | center | [查看] [分配] | - |

**任务状态Tag映射：**
| 状态值 | 显示文本 | Tag颜色 |
|--------|---------|---------|
| `pending` | 待执行 | `blue` |
| `in_progress` | 执行中 | `orange` |
| `completed` | 已完成 | `green` |

**特殊行样式：**
- 已逾期未完成任务：行背景浅红色 `#fff1f0` + 红色标签"已逾期"，置顶
- 即将到期（24小时内）未完成任务：黄色标签"即将到期"

**操作列详情：**
| 操作 | 类型 | 权限 | 行为 | 条件显示 |
|------|------|------|------|---------|
| 查看 | Link | all | 跳转 `/operations/inspection/tasks/:id` | 始终显示 |
| 分配 | Link | `safety_officer` | 打开执行人选择弹窗 | 状态为待执行时显示 |

**列表排序：** 默认按"逾期置顶 + 状态优先(待执行→执行中→已完成) + 截止日期升序"
**分页：** 默认 20 条/页，可选 10/20/50/100

#### 分配执行人弹窗 (Modal)

| 字段 | 标签 | 组件 | 必填 | 说明 |
|------|------|------|------|------|
| `assigneeId` | 执行人 | Select(带搜索) | 是 | 搜索员工姓名/工号，展示姓名+角色 |

**弹窗行为：**
| 触发 | 行为 |
|------|------|
| 确认 | 分配执行人 → Toast"已分配给XXX" → 刷新列表 |
| 取消 | 关闭弹窗 |

#### 交互行为

| 触发 | 行为 | 路由路径 | 说明 |
|------|------|----------|------|
| 点击任务编号 | 跳转详情 | `/operations/inspection/tasks/:id` | id来自当前行 |
| 点击计划名称 | 跳转计划详情 | `/operations/inspection/plans/:planId` | planId来自当前行 |
| 点击统计卡片数字 | 筛选列表 | - | 更新筛选条件 |
| 切换状态Tab | 刷新列表 | - | 按状态筛选 |
| 点击 [分配] | 打开分配Modal | - | 选择执行人 |

#### 空状态

```
暂无安检任务，请先创建安检计划并新增任务
[新增任务]  [前往安检计划]
```
按钮跳转 `/operations/inspection/plans`

#### 响应式适配

| 断点 | 布局调整 |
|------|---------|
| ≥1200px | 统计卡5列，完整表格视图 |
| 768-1199px | 统计卡3+2列，隐藏次要列(关联计划) |
| <768px | 统计卡纵向堆叠2列，卡片列表视图（编号/状态/进度前置） |

### P02: 安检任务详情/执行页 (InspectionTaskExecution)

**路由：** `/operations/inspection/tasks/:id`
**对应 Story：** US-008, US-014-A
**权限：** `safety_officer`, `station_master`, `inspector`

#### 页面头部

- **面包屑：** 首页 / 基础运营 / 巡检/安检管理 / 安检任务 / 执行巡检
- **标题：** 执行巡检
- **操作按钮：**
  - `[返回列表]` → 跳转 `/operations/inspection/tasks`

#### 固定顶部进度条区域

```
┌─────────────────────────────────────────────────────────────────┐
│ 任务: IT-HZXH-0219-001  计划: 2月日常巡检  执行人: 张三        │
│ 截止: 2026-02-19        状态: 🟠 执行中                        │
├─────────────────────────────────────────────────────────────────┤
│ 进度: 5/12              ████████░░░░ 42%                       │
└─────────────────────────────────────────────────────────────────┘
```

**任务信息字段：**

| 字段 | 标签 | 渲染 |
|------|------|------|
| `taskNo` | 任务 | 文本(加粗) |
| `plan.name` | 计划 | 链接→P05 |
| `assignee.name` | 执行人 | 文本 |
| `dueDate` | 截止 | 日期，逾期红色 |
| `status` | 状态 | StatusTag |
| `checkedItems/totalItems` | 进度 | Progress进度条 + "X/Y" + 百分比 |

#### 检查项目分组列表

**布局：** 按检查项 `category` 分类分组，使用 Collapse 组件，默认全部展开

**分类组标题：**
```
▎罐区环保 (2/3)                              [全部正常 ✓]
```

| 元素 | 说明 |
|------|------|
| 分类名称 | 中文分类名 |
| 完成统计 | "已检查数/该分类总数" |
| 全部正常按钮 | 仅在该组有未检查项时显示；任务已完成时隐藏 |

**检查项分类中文映射：**
| category值 | 中文名 |
|------------|--------|
| `tank_area` | 罐区环保 |
| `dispenser` | 加气机 |
| `power_room` | 配电室 |
| `fueling_area` | 加油区域 |
| `non_fuel` | 非油管理 |
| `equipment` | 设备管理 |

**单个检查项行：**

```
┌─────────────────────────────────────────────────────────────┐
│ ● LNG储罐#1外观检查                   [✓正常] [✗异常]      │
│   检查标准: 检查储罐外壁有无凹陷、腐蚀...                    │
│   状态: ✅ 正常  08:30                                      │
├─────────────────────────────────────────────────────────────┤
│ ● 加气机#02运行检查                    [✓正常] [✗异常]      │
│   检查标准: 检查加气枪...                                    │
│   状态: 🔴 异常  08:45                                      │
│   ┌───────────────────────────────────────────────────────┐ │
│   │ 备注 *: 加气枪#2有微量泄漏                             │ │
│   │ 📷 [上传照片] leak.jpg ✕                               │ │
│   │ [登记问题→]                                            │ │
│   └───────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────┤
│ ○ 罐区防泄漏设施检查                   [✓正常] [✗异常]      │
│   检查标准: 检查泄漏报警器...                                │
│   状态: ⏳ 待检查                                           │
└─────────────────────────────────────────────────────────────┘
```

**检查项字段：**

| 字段 | 渲染 | 说明 |
|------|------|------|
| `checkItem.name` | 文本(加粗) | 检查项名称 |
| `checkItem.description` | 次要文字 | 检查标准描述，默认收起，悬浮展示或点击展开 |
| `result` | 状态图标+文字+时间 | ⏳待检查(灰) / ✅正常(绿) / 🔴异常(红) |
| `executedAt` | 文本 | 结果标记时间 HH:mm |
| `remark` | TextArea | 异常时展开，必填 |
| 上传照片 | Upload | [MVP+] 异常区域内，最多10张 |
| 登记问题 | Link按钮 | 异常区域内，点击打开P12登记问题抽屉 |

**操作按钮：**

| 按钮 | 组件 | 样式 | 行为 |
|------|------|------|------|
| ✓正常 | Button | type="primary", `#52c41a`背景 | 标记正常→即时保存→自动滚动到下一个待检查项 |
| ✗异常 | Button | type="primary" danger | 标记异常→展开备注区域→焦点到备注输入框 |
| 全部正常 | Button | type="default" | 批量标记该分类下所有未检查项为正常→Toast"X项已标记为正常" |

**操作按钮状态：** 任务状态为 `completed` 时，所有操作按钮隐藏，页面为纯查看模式

#### 底部固定操作栏 (Affix)

```
┌─────────────────────────────────────────────────────────────────┐
│                              还有 7 项未检查   [提交完成]（灰显）│
└─────────────────────────────────────────────────────────────────┘
```

| 状态 | 按钮显示 |
|------|---------|
| 有未检查项 | [提交完成] 灰显 disabled，旁边提示"还有N项未检查" |
| 全部已检查 | [提交完成] 绿色高亮 enabled |
| 任务已完成 | 不显示底部操作栏 |

#### 交互行为

| 触发 | 行为 | 说明 |
|------|------|------|
| 点击 [✓正常] | 标记→保存→进度+1→滚动到下一项 | 首次操作自动将任务状态改为执行中 |
| 点击 [✗异常] | 标记→展开备注→进度+1 | 备注必填，实时保存 |
| 点击 [全部正常] | 批量标记 → Toast | 确认弹窗(Popconfirm):"将该分类下N项标记为正常？" |
| 点击已标记项的按钮 | 切换结果/重新标记 | 支持修改（正常↔异常），标记后5秒内Toast提供撤销 |
| 点击 [登记问题→] | 打开P12抽屉 | 自动预填关联任务、检查项目、关联设备 |
| 点击 [提交完成] | 确认弹窗→提交→任务状态→已完成 | Toast"巡检任务已完成" → 返回任务列表 |
| 点击计划名称 | 跳转计划详情 | `/operations/inspection/plans/:planId` |
| 点击 [返回列表] | 返回任务列表 | `/operations/inspection/tasks` |

#### 首次使用引导 (一次性)

首次进入执行页时浮窗提示：
```
💡 点击 ✓正常 或 ✗异常 逐项记录检查结果，也可点击「全部正常」批量标记
```
关闭后写入 localStorage，不再显示。

#### 响应式适配

| 断点 | 布局调整 |
|------|---------|
| ≥1200px | 固定顶部进度条+分组列表+底部操作栏 |
| <768px | 任务信息紧凑化，检查项大按钮保持（适合触摸），底部操作栏固定 |
