## Epic 1: 设施监控

来源功能点：F001, F002, F003, F004, F005, F006, F007

### US-001: 查看设施监控看板

**优先级：** [MVP]
**功能点：** F001
**角色：** 站长 / 设备管理员

> 作为站长，
> 我希望以看板形式查看站点所有关键设施的实时运行状态概览，
> 以便随时掌握站点设备的整体运行情况。

**验收标准：**

1. 假设当前站点有已录入的设备数据，当进入设施监控看板页面，那么以卡片/面板形式展示各类设施的运行状态概览（储罐区、加气机区、关键设备区）。
2. 假设站点有多个储罐，当查看看板，那么储罐区域显示每个储罐的液位百分比、压力指示以及正常/异常状态标识。
3. 假设站点有多台加气机，当查看看板，那么加气机区域显示每台加气机的当前状态（空闲/加注中/故障/离线）及枪位状态。
4. 假设某设备状态异常（如储罐压力过高），当查看看板，那么对应设备卡片以红色/警告色标识，展示异常摘要信息。
5. 假设用户需要查看更多细节，当点击某设备卡片，那么跳转到对应设备的详细监控页面。

**关联：** US-002, US-003
**前置依赖：** 设备台账已录入（US-006）

### US-002: 查看储罐液位与压力

**优先级：** [MVP]
**功能点：** F002
**角色：** 设备管理员 / 安全员

> 作为设备管理员，
> 我希望查看每个储罐的当前液位、压力和温度数据，
> 以便监控储罐运行状态，确保安全和库存充足。

**验收标准：**

1. 假设站点有已配置的储罐，当进入储罐监控页面，那么展示每个储罐的名称/编号、液位（百分比和立方米）、压力值（MPa）、温度（℃）。
2. 假设储罐液位数据存在历史记录，当查看某储罐详情，那么显示近 24 小时的液位变化趋势图。
3. 假设储罐液位低于安全阈值（如 20%），当查看液位数据，那么该储罐卡片以警告色标识，并显示"液位过低"提示。
4. 假设储罐压力超过安全阈值，当查看压力数据，那么该储罐卡片以错误色标识，并显示"压力异常"提示。

**关联：** US-001, US-005
**前置依赖：** US-001

### US-003: 查看加气机状态

**优先级：** [MVP]
**功能点：** F003
**角色：** 设备管理员 / 站长

> 作为站长，
> 我希望查看站点每台加气机及其枪位的实时状态，
> 以便了解加气机的可用情况，协调现场运营。

**验收标准：**

1. 假设站点有已配置的加气机，当进入加气机状态页面，那么以卡片或列表形式展示每台加气机的编号、型号、当前状态（空闲/加注中/故障/离线）。
2. 假设某加气机有多个枪位，当查看该加气机详情，那么显示每个枪位的状态、燃料类型、当日充装量。
3. 假设某加气机处于故障状态，当查看加气机列表，那么该加气机以红色标识，显示故障时间和故障描述。
4. 假设某加气机处于离线状态，当查看加气机列表，那么该加气机以灰色标识，显示离线时间。

**关联：** US-001, 站点管理模块枪配置
**前置依赖：** US-001

### US-004: 查看传感器数据

**优先级：** [MVP+]
**功能点：** F004
**角色：** 安全员 / 设备管理员

> 作为安全员，
> 我希望查看站点所有已安装传感器的实时读数，
> 以便及时发现环境异常，保障安全。

**验收标准：**

1. 假设站点有已配置的传感器，当进入传感器数据页面，那么以列表/卡片形式展示各传感器的名称、位置、当前读数、单位、状态。
2. 假设传感器有历史数据，当点击某传感器卡片，那么展示该传感器近 24 小时的数据趋势图。
3. 假设某传感器读数超出正常范围（如可燃气体浓度 > 25% LEL），当查看数据，那么以警告色标识并显示告警信息。
4. 假设某传感器离线，当查看列表，那么该传感器以灰色标识，显示"离线"状态和最后在线时间。

**关联：** US-001, US-005
**前置依赖：** US-001

### US-006-A: 配置告警阈值

**优先级：** [MVP+]
**功能点：** F006
**角色：** 设备管理员 / 站长

> 作为设备管理员，
> 我希望配置关键指标的告警阈值，
> 以便系统在指标异常时自动发出告警通知。

**验收标准：**

1. 假设在设施监控页面，当点击"告警配置"入口，那么显示告警规则配置页面。
2. 假设需要配置储罐液位告警，当设置"液位低于 20% 时告警"并保存，那么配置生效，后续符合条件时系统生成告警记录。
3. 假设已有告警规则，当查看告警配置列表，那么显示所有已配置的告警规则（设备/指标/阈值/告警级别/启用状态）。
4. 假设需要调整阈值，当编辑已有规则并保存，那么新阈值立即生效。

**关联：** US-006-B
**前置依赖：** US-001

### US-006-B: 查看告警记录

**优先级：** [MVP+]
**功能点：** F007
**角色：** 站长 / 安全员

> 作为站长，
> 我希望查看历史和当前的告警事件记录，
> 以便追踪设备异常历史，评估设备安全状态。

**验收标准：**

1. 假设系统已产生告警记录，当进入告警记录页面，那么以列表形式展示告警事件，包含告警时间、设备名称、告警类型、告警级别、当前状态（未处理/已确认/已解决）。
2. 假设告警数量较多，当使用筛选条件（设备、告警级别、时间范围、状态），那么列表按条件过滤展示。
3. 假设有未处理的告警，当查看列表，那么未处理告警排在最前，以醒目标识显示。
4. 假设需要确认告警，当点击"确认"按钮并填写处理说明，那么告警状态变为"已确认"。

**关联：** US-006-A
**前置依赖：** US-006-A

### US-007: 查看设备台账列表

**优先级：** [MVP]
**功能点：** F009, F016, F017
**角色：** 设备管理员

> 作为设备管理员，
> 我希望以列表形式查看站点所有设备的台账概要信息，
> 以便快速了解站点设备的总体情况。

**验收标准：**

1. 假设站点有已录入的设备数据，当进入设备台账页面，那么显示设备列表，包含设备编号、名称、类型、型号、状态、安装日期列。
2. 假设设备数量超过单页显示数量，当翻页，那么正确显示对应页的设备数据。
3. 假设需要查找特定设备，当在搜索框输入设备编号或名称，那么列表实时过滤显示匹配的设备。
4. 假设需要按条件筛选，当选择"设备类型"、"状态"或"安装日期范围"筛选条件，那么列表只显示符合条件的设备。
5. 假设需要按设备类型分组查看，当选择"按类型分组"视图，那么列表按设备类型（储罐/加气机/泵/阀门/传感器等）分组展示。

**关联：** US-008, US-009, US-010
**前置依赖：** 无

### US-008: 新增设备台账

**优先级：** [MVP]
**功能点：** F010, F014
**角色：** 设备管理员

> 作为设备管理员，
> 我希望录入新设备的台账信息，
> 以便将新安装的设备纳入系统管理。

**验收标准：**

1. 假设在设备台账列表页，当点击"新增设备"按钮，那么打开新增设备表单页面。
2. 假设填写表单时，当输入设备编号，那么系统实时校验编号的全局唯一性，重复则提示"设备编号已存在"。
3. 假设完整填写设备编号、名称、类型、型号、制造商、安装日期、维保周期等必填字段，当点击"保存"，那么设备创建成功并跳转到设备列表，新设备出现在列表中。
4. 假设必填字段未填写完整，当点击"保存"，那么表单提示必填字段错误，不允许提交。
5. 假设需要选择设备类型，当点击"设备类型"下拉，那么显示预定义的设备类型列表（储罐、加气机、泵、阀门、传感器、消防设备、电气设备等）。
6. 假设需要设置维保周期，当选择维保频率（日检/周检/月检/季检/半年检/年检），那么系统自动计算下次维保日期。

**关联：** US-007
**前置依赖：** 无

**优先级：** [MVP]
**功能点：** F011
**角色：** 设备管理员

4. 假设需要查看已停用的设备，当在筛选条件中选择"显示停用设备"，那么列表中显示已停用的设备（灰显或带停用标记）。

### US-015: 创建维保工单

**优先级：** [MVP]
**功能点：** F022
**角色：** 设备管理员

> 作为设备管理员，
> 我希望手动创建维保工单，
> 以便安排设备的保养或维修任务。

**验收标准：**

1. 假设在维保工单列表页，当点击"新建工单"按钮，那么打开工单创建表单。
2. 假设填写工单信息，当选择设备（从设备台账中选择），那么自动带出设备编号、名称、型号等基本信息。
3. 假设完整填写工单类型（保养/维修）、问题描述、计划时间、指派负责人，当点击"保存"，那么工单创建成功，状态为"待处理"，显示在工单列表中。
4. 假设必填字段未填写完整，当点击"保存"，那么表单提示必填字段错误，不允许提交。
5. 假设从设备详情页，当点击"创建维保工单"快捷入口，那么自动预填设备信息，进入工单创建表单。

### US-016: 查看工单详情与状态流转

**优先级：** [MVP]
**功能点：** F024, F025
**角色：** 设备管理员 / 站长

> 作为设备管理员，
> 我希望查看维保工单的详细信息并推进工单状态流转，
> 以便跟踪维保任务的完成进度。

**验收标准：**

1. 假设在工单列表页，当点击某工单的编号或"查看详情"，那么进入工单详情页。
2. 假设进入工单详情页，当页面加载完成，那么显示工单基本信息（编号、类型、设备信息、问题描述）、状态时间线、负责人、计划时间、处理记录。
3. 假设工单状态为"待处理"，当具有权限的负责人点击"开始处理"，那么工单状态变为"处理中"，记录开始处理时间。
4. 假设工单状态为"处理中"，当负责人填写处理结果并点击"提交验收"，那么工单状态变为"待验收"。
5. 假设工单状态为"待验收"，当站长或技术主管点击"验收通过"，那么工单状态变为"已完成"，记录完成时间。
6. 假设验收不通过，当点击"退回处理"并填写原因，那么工单状态退回"处理中"，负责人收到通知。6. 假设工单需要审批，当工单流转到需审批节点，那么调用系统统一审批流模块，按站点配置的维保工单审批流执行审批。
**关联：** US-014, US-015, US-019
**前置依赖：** US-015

5. 假设紧急程度为"紧急"，当提交报修后，那么工单以醒目标识展示在工单列表最前方。
