## 1. 用户角色分析

### 1.1 设备管理员 (equipment_admin)

**基本信息：**
- 典型画像：28-40岁，持特种设备作业证（压力容器、压力管道），设备管理专业背景
- 工作环境：站点设备区为主，需频繁在设备现场与办公室之间切换
- 使用频率：每日高频（10-20次/天），维保期间更频繁
- 技术熟练度：中等，擅长设备操作，对软件系统一般熟悉

**核心诉求：**
- 主要目标：高效维护设备台账数据，管理维保工单，确保设备运行安全
- 痛点：
  - 设备台账信息录入字段多，特别是新设备批量入库时效率低
  - 维保工单状态散落在不同页面，难以跟踪处理进度
  - 在设备现场发现故障时，需要快速提交报修但操作步骤多
  - 不确定哪些设备即将到期需要保养
- 期望：简化台账录入流程，工单进度一目了然，一键快速报修，维保日程自动提醒

**场景约束：**
- 时间压力：设备故障时压力极大，需快速响应报修
- 注意力分散：高，需同时关注现场多台设备运行状况
- 专业知识：深厚的设备技术知识，理解储罐/加气机/传感器各类参数含义

### 1.2 站长 (station_master)

**基本信息：**
- 典型画像：30-45岁，持危险化学品经营许可证负责人资格证，站点全面管理者
- 工作环境：站点现场与办公室交替，巡视设备区
- 使用频率：每日中频（5-10次/天）
- 技术熟练度：中等，熟悉站点所有业务流程

**核心诉求：**
- 主要目标：总览设备运行态势，审批维保工单，确认验收维保结果
- 痛点：
  - 需要在不同页面间切换才能了解站点设备全貌（储罐、加气机、传感器）
  - 维保工单的审批验收入口不够醒目，有时遗漏待处理的审批项
  - 设备故障对生产的影响难以快速评估
- 期望：一屏总览设备状况，待办审批醒目提醒，从监控看板直达工单详情

**场景约束：**
- 时间压力：高，现场事务繁多，操作常被打断
- 注意力分散：高，经常在处理一个任务时被其他事务中断
- 专业知识：了解设备运行参数但侧重管理决策

---

### 1.3 技术负责人 (tech_lead)

**基本信息：**
- 典型画像：35-50岁，深冷/工艺类工程背景，负责设备维保方案和技术审核
- 工作环境：站点技术办公室为主，定期到设备区指导
- 使用频率：每日中频（3-8次/天）
- 技术熟练度：中等偏高，熟悉工程技术系统

**核心诉求：**
- 主要目标：制定维保方案，审核维修记录，处理疑难故障
- 痛点：
  - 查看维保记录溯源时需要逐个打开工单详情
  - 设备健康度评估缺乏直观指标
  - 保养计划制定后缺乏系统性跟踪工具
- 期望：设备维保时间线一览，健康度可视化评分，保养计划日历视图

**场景约束：**
- 时间压力：中等，日常较为从容，紧急故障时压力大
- 注意力分散：低至中等，能较集中地处理技术问题
- 专业知识：最深厚的设备专业知识，理解所有参数和工艺流程

### 1.4 安全员 (safety_officer)

**基本信息：**
- 典型画像：25-40岁，持注册安全工程师或安全管理人员证书
- 工作环境：站点现场巡查为主，安全监控室
- 使用频率：每日中频（5-10次/天），主要关注告警和安全指标
- 技术熟练度：中等，理解安全参数含义

**核心诉求：**
- 主要目标：监控设备安全参数，及时响应告警，确认设备安全状态
- 痛点：
  - 传感器数据（可燃气体、温度、压力）分散在不同区域，无法一屏监控
  - 告警信息容易被淹没在大量正常数据中
  - 故障报修时需要填写的信息太多，影响响应速度
- 期望：安全参数集中展示，告警醒目突出，快速报修入口

**场景约束：**
- 时间压力：告警触发时极高，需要秒级响应
- 注意力分散：中等，需要在巡查和监控间切换
- 专业知识：安全参数阈值判断，应急处理流程

---

### 1.5 运营经理 (ops_manager)

**基本信息：**
- 典型画像：35-50岁，管理多个站点的运营决策者
- 工作环境：总部办公室，偶尔巡视站点
- 使用频率：每日低至中频（2-5次/天）
- 技术熟练度：中等偏高，熟悉管理系统

**核心诉求：**
- 主要目标：跨站点设备健康概览，设备资产总览，维保投入分析
- 痛点：
  - 各站点设备数据分散，无法快速横向对比
  - 不了解各站点设备维保执行情况
- 期望：多站点设备健康度横向对比，维保工单完成率统计

**场景约束：**
- 时间压力：中等，主要做决策而非执行
- 注意力分散：中等，需同时管理多个模块
- 专业知识：理解设备管理指标，侧重经营和投入产出

### 任务 1：查看设施监控看板 (US-001)

**用户目标：** 一屏总览站点所有关键设施的实时运行状态
**使用频率：** 每日10+次（站长/设备管理员开始巡视前必看）
**时间压力：** 高

**优化后流程：**

```
[进入设施监控] → [扫视看板全貌] → [关注异常区域] → [点击设备卡片深入查看]
       ↓                ↓               ↓                    ↓
  默认首页展示    三大区域按重要性排列   异常红色/警告黄色突出   跳转到对应详情页
```

**具体设计决策：**

| 步骤 | 用户行为 | 设计决策 | 依据 |
|------|---------|---------|------|
| 1 | 进入监控页 | 进入设备设施管理模块时，默认落地页即为设施监控看板 | 监控是最高频操作，减少导航点击 |
| 2 | 扫视全貌 | 看板分三区域：储罐区（液位/压力卡片）、加气机区（状态卡片）、关键指标汇总（设备总数/在线率/告警数/待维保数） | 按重要性分区，视觉层次清晰 |
| 3 | 识别异常 | 异常设备卡片红色边框+图标闪动，告警数量Badge在区域标题旁 | 人眼对红色和动态元素最敏感 |
| 4 | 深入查看 | 点击设备卡片直接跳转对应详情页 | 减少中间步骤 |

```
[看板点击储罐卡片] → [查看储罐详情] → [查看趋势图] → [发现异常→报修]
```

### 任务 3：查看加气机状态 (US-003)

**用户目标：** 掌握每台加气机及枪位的实时状态和可用情况
**使用频率：** 每日5-10次
**时间压力：** 中等

**优化后流程：**

```
[看板点击加气机区域] → [查看加气机列表/看板] → [点击查看枪位详情]
         ↓                    ↓                       ↓
    1次点击可达         按状态色标直观展示         枪位状态+当日充装量
```

**具体设计决策：**

| 步骤 | 用户行为 | 设计决策 | 依据 |
|------|---------|---------|------|
| 1 | 查看加气机 | 以卡片网格展示，一台加气机一张卡片 | 卡片布局便于快速扫视状态 |
| 2 | 识别状态 | 绿色=空闲，蓝色=加注中，红色=故障，灰色=离线：大色块显示 | 颜色编码是最快速的状态识别方式 |
| 3 | 故障加气机 | 故障卡片置顶显示，红色边框+闪烁图标 | 故障设备最需关注 |
| 4 | 查看枪位 | 卡片内直接展示枪位状态列表，无需进入详情 | 减少层级降低获取信息成本 |

| 步骤 | 用户行为 | 设计决策 | 依据 |
|------|---------|---------|------|
| 1 | 发起新增 | 列表页右上角"新增设备"按钮 | 符合用户习惯，显眼可达 |
| 2 | 选择类型 | 表单首栏选择设备类型，选择后动态展示该类型特有字段 | 不同类型设备的必填字段不同（如储罐有容量、加气机有枪位数） |
| 3 | 填写编号 | 默认"自动生成"并预填建议编号（如 DEV-TANK-001），可切换手工输入；输入时实时校验全局唯一性 | 降低编码规则学习成本，防止重复 |
| 4 | 信息分组 | 表单分组：基础信息（必填）→ 技术参数 → 维保设置 → 备注 | 渐进式披露，必填优先 |
| 5 | 维保周期 | 选择频率后系统自动计算下次维保日期 | 减少手动计算，降低出错概率 |
| 6 | 保存结果 | 保存成功后跳转设备详情页，同时显示"复制并创建下一条"按钮 | 引导用户确认录入结果；连续录入时可一键复制当前设备信息创建新设备 |

## 6. 错误预防与恢复

### 6.1 错误预防

| 操作 | 潜在错误 | 预防措施 |
|------|---------|---------|
| 新增设备：编号输入 | 编号重复或格式不规范 | 提供"自动生成"默认选项；手工输入时实时校验全局唯一性，重复则即时提示 |
| 新增设备：必填遗漏 | 遗漏必填字段导致保存失败 | 必填字段红色星号标识；保存时高亮所有未填必填项并定位到第一个错误字段 |
| 编辑设备：未保存离开 | 修改数据丢失 | 检测修改→离开时弹窗确认；自动保存草稿到本地 |
| 删除设备 | 误删除在用设备 | 二次确认弹窗，显示关联数据数量（如"该设备有3条未完成的维保工单"）；有进行中工单时禁止删除 |
| 创建工单：设备选择 | 选错设备 | 设备选择器带搜索，选中后显示设备编号+名称+类型确认 |
| 工单状态推进 | 误操作推进（如误点"验收通过"） | 状态变更前弹窗确认并要求填写说明 |
| 故障报修：紧急程度 | 选错紧急程度 | 各级别带明确说明文字，提交后5秒内可撤销修改 |
| 批量操作（导出） | 导出范围超出预期 | 导出前显示"将导出X条记录"的预览提示 |
| 表单中途离开 | 浏览器关闭/刷新导致数据丢失 | 表单数据自动保存到浏览器本地存储，下次打开恢复 |

### 6.2 错误恢复

| 错误场景 | 恢复方案 |
|---------|---------|
| 误删除设备 | 软删除机制→在"停用设备"筛选中可查看，支持"恢复启用"操作 |
| 误删除设备照片 | 操作后5秒内显示"撤销"Toast链接 |
| 误提交工单状态 | 待验收状态下站长可"退回处理"；已完成工单可由管理员重新打开 |
| 表单填写中断 | 自动保存草稿到浏览器本地存储，下次进入同一表单恢复 |
| 网络断开 | 提示"网络已断开"，监控看板显示最后更新时间；编辑中的数据本地缓存，网络恢复后可提交 |
| 报修信息不完整 | 提交报修后可在工单详情页补充完善信息 |

## 11. 响应式设计要点

基于站点管理模块的确认（站长需要在现场用手机/平板操作），设备设施管理同样需要响应式支持。设备管理员和安全员在设备现场使用移动设备的场景更为典型。

### 11.1 断点定义

| 断点 | 宽度 | 目标设备 | 布局调整 |
|------|------|---------|---------|
| 桌面端 | ≥1200px | PC 显示器 | 完整侧边栏 + 内容区 |
| 平板端 | 768-1199px | iPad/平板 | 折叠侧边栏 + 简化看板 |
| 移动端 | <768px | 手机 | 底部导航 + 卡片列表 |

### 11.2 关键页面移动端适配

| 页面 | 桌面布局 | 移动端布局 |
|------|---------|-----------|
| 设施监控看板 | 三区域并列（储罐/加气机/待处理） | 纵向堆叠，关键指标汇总前置 |
| 储罐监控 | 储罐卡片横向排列 + 趋势图 | 储罐卡片纵向堆叠，趋势图全宽 |
| 加气机状态 | 网格卡片布局 | 纵向卡片列表 |
| 设备台账列表 | 表格视图 | 卡片列表，关键信息（编号/名称/状态）前置 |
| 设备详情 | 多Tab并列 | Tab下拉或滑动切换 |
| 维保工单列表 | 表格视图 | 卡片列表，状态色标突出 |

## 12. 动效设计

### 12.1 动效原则

- **有意义**：每个动效服务于提示或引导目的，不纯粹装饰
- **不阻塞**：动效不阻塞用户操作，加载中状态骨架屏而非全屏Loading
- **一致性**：遵循 Fluent Design System 动效token

### 12.2 动效场景

| 场景 | 动效类型 | 时长 | 说明 |
|------|---------|------|------|
| 页面切换 | Fade + Slide | 200ms | 页面内容淡入+轻微滑入 |
| 看板数据刷新 | 数值过渡 | 300ms | 数值变化平滑过渡，非跳变 |
| 异常设备闪烁 | 边框闪烁 | 1s周期 | 故障设备卡片红色边框慢闪 |
| 抽屉打开（报修） | Slide from right | 250ms | 右侧滑入 |
| Toast 提示 | Slide down + Fade | 200ms入/3s停留/200ms出 | 顶部滑入-停留-淡出 |
| 状态标签变更 | Scale + Color | 300ms | 标签颜色变化+轻微缩放反馈 |
| 工单步骤条推进 | Progress fill | 400ms | 步骤间连线填充动画 |
| 液位条变化 | Height transition | 500ms | 液位高度平滑变化 |
