## 1. 模块概述

### 1.1 业务目标

设备设施管理是加气站运营的基础支撑模块，确保：
- 站点所有设备建立完整台账档案，支持全生命周期管理
- 储罐、加气机等关键设施的实时状态可视化监控
- 维保计划的制定、执行和闭环跟踪
- 设备联网对接，实现数据自动采集与状态同步

### 1.2 模块边界

| 范围 | 说明 |
|------|------|
| **包含** | 设施监控（储罐液位/压力看板、加气机状态、传感器数据、设备健康度）、设备台账（设备档案、型号、安装日期、维保周期、设备照片）、维保与工单（保养计划、维修记录、故障报修与闭环、维保提醒）、设备联网对接（加注机数据采集、油枪信息读取、实时状态监控） |
| **不包含** | 枪配置管理（属于站点管理 1.1）、巡检计划与执行（属于巡检/安检管理 1.4）、库存管理（属于能源交易 2.3） |

### 1.3 与其他模块关系

| 关联模块 | 关系说明 |
|----------|----------|
| 站点管理 (1.1) | 设备归属站点；枪配置的状态数据来源于本模块的数据采集 |
| 巡检/安检管理 (1.4) | 设备台账与巡检"检查项目"自动关联；维保问题可同步至巡检问题记录 |
| 能源交易 > 库存管理 (2.3) | 储罐液位数据为库存管理的数据来源之一 |
| 数据分析 (7.1) | 设备运行数据为经营看板和多维分析提供数据支撑 |

## 2. 功能拆解清单

### 2.1 设施监控（来源：需求文档 §1.3 表格第1行）

**功能描述：** 储罐液位/压力、加气机状态、传感器数据、设备健康度。可扩展：阈值告警、维保工单自动触发；油罐液位监控、安全预警。

| # | 功能点（L4 原子功能） | 描述 | 优先级 | 来源 |
|---|---------------------|------|--------|------|
| F001 | 查看设施监控看板 | 以看板形式展示站点所有关键设施（储罐、加气机等）的实时运行状态概览 | [MVP] | §1.3 行1 |
| F002 | 查看储罐液位与压力 | 展示每个储罐的当前液位（百分比、立方米）、压力值、温度，支持液位变化趋势图 | [MVP] | §1.3 行1 |
| F003 | 查看加气机状态 | 展示每台加气机及其枪位的实时状态（空闲/加注中/故障/离线） | [MVP] | §1.3 行1 |
| F004 | 查看传感器数据 | 展示站点已安装传感器的实时读数（温度、压力、可燃气体浓度等），支持数据图表展示 | [MVP+] | §1.3 行1 |
| F005 | 查看设备健康度概览 | 以健康度评分或状态指示器形式展示关键设备的综合健康状态 | [MVP+] | §1.3 行1 |
| F006 | 阈值告警配置 | 配置关键指标的告警阈值（如储罐液位低于 20%、压力超过设定值），触发告警通知 | [MVP+] | §1.3 行1 说明 |
| F007 | 查看告警记录 | 展示历史和当前的告警事件列表，支持按设备、告警级别、时间筛选 | [MVP+] | §1.3 行1 说明 |
| F008 | 维保工单自动触发 | 当设备监控指标异常或达到维保周期时，自动创建维保工单 | [PROD] | §1.3 行1 说明 |

## 3. 依赖关系

### 3.1 模块内依赖

- F010（新增设备台账）→ F011, F012, F013: 需先有设备才能编辑、查看详情、删除
- F010（新增设备台账）→ F001-F005: 监控看板依赖设备台账存在
- F010（新增设备台账）→ F021-F032: 维保工单依赖设备台账存在
- F022（创建维保工单）→ F023, F024, F025, F029: 工单生命周期依赖工单存在
- F027（制定保养计划）→ F028: 维保提醒依赖保养计划
- F006（阈值告警配置）→ F007: 告警记录依赖告警配置

### 3.2 跨模块依赖

| 功能点 | 依赖模块 | 依赖说明 |
|--------|----------|----------|
| F001（设施监控看板） | 站点管理 (1.1) | 需要站点上下文，监控数据按站点展示 |
| F003（加气机状态） | 站点管理 (1.1) | 加气机/枪配置数据来源于站点管理模块 |
| F018（台账与巡检关联） | 巡检/安检管理 (1.4) | 设备台账变更需同步巡检"检查项目" |

## 4. 疑问确认记录

**确认日期：** 2026-02-18

| # | 问题 | 用户确认结果 |
|---|------|-------------|
| Q1 | 设备类型分类体系？ | **固定分类**：储罐、加气机、泵、阀门、传感器、消防设备、电气设备 |
| Q2 | 设备照片数量/格式/大小限制？ | 每设备最多 10 张，格式 JPG/PNG，单张不超过 5MB |
| Q3 | 设备编号唯一性范围？ | **全局唯一** |
| Q4 | 台账删除方式？ | **软删除**：标记为停用，保留历史数据 |
| Q5 | 维保工单的审批流程？ | **含审批**：审批流由系统统一模块设计，维保工单模块调用系统审批流功能并建立适合维保工单的审批流应用；每个站可有不同审批流 |
| Q6 | 故障报修的紧急程度分几级？ | **4 级**：低/中/高/紧急 |
| Q7 | 设施监控数据在 MVP 阶段如何模拟？ | 使用静态模拟数据 + 随机波动，模拟实时监控效果 |
| Q8 | 设备联网对接在 MVP 阶段如何处理？ | 仅展示连接状态列表（模拟数据），采集配置标记为 [PROD] |
| Q9 | 台账与巡检项目的自动同步机制？ | MVP 阶段预留接口但不实现，标记为 [PROD] |

## 7. 参考的角色与职能

基于 `knowledge-base/LNG加气站核心人员职能及协作流程.md`，设备设施管理模块涉及的主要角色：

| 深冷维修工 | 低温设备的专业维修执行，LNG 特有维保操作 |
| 工艺技术员 | 储罐、气化器等工艺设备的监控和维护 |
