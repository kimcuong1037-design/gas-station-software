## 3. 路由一致性专项检查（新增）

### 检查方法

```bash
# 1. 提取路由定义
grep -r "path:" src/router.tsx

# 2. 提取导航调用
grep -rE "navigate\(|to=" src/features/ --include="*.tsx" | grep -E "station|operations"
```

### 检查结果

| # | 检查项 | 状态 | 说明 |
|---|--------|------|------|
| R-1 | 路由定义一致性 | ✅ | router.tsx 定义与 UI Schema 一致 |
| R-2 | 导航链接有效性 | ✅ | 所有 `navigate()` 指向已定义路由 |
| R-3 | 无硬编码路径错误 | ✅ | 已修复 `/create` → `/new` |
| R-4 | 动态路由匹配顺序 | ✅ | `new` 在 `:id` 之前定义 |
| R-5 | 404 处理 | ⚠️ | 未定义全局 404 页面（建议改进） |

### 路由映射表

| 路由路径 | 页面组件 | 导航来源 | 状态 |
|---------|---------|---------|------|
| `/operations/station` | StationList | 侧边栏、面包屑、返回按钮 | ✅ |
| `/operations/station/new` | StationForm | 新增按钮 | ✅ 已修复 |
| `/operations/station/:id` | StationDetail | 列表行点击、卡片点击 | ✅ |
| `/operations/station/:id/edit` | StationForm | 编辑按钮 | ✅ |

### 修复记录

**问题**: 点击"新增站点"按钮显示"站点不存在"  
**原因**: 按钮跳转路径 `/create` 与路由定义 `/new` 不一致，被 `:id` 动态路由捕获  
**修复**: `navigate('/operations/station/create')` → `navigate('/operations/station/new')`
