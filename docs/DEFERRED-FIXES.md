# 延期修复清单 (Deferred Fixes)

**用途：** 记录在某个 Phase/模块确认之后发现的、需要回补修改的问题。这些问题不影响当前阶段的交付，但需要在未来合适的时间点统一修复。
**创建日期：** 2026-02-24
**版本：** 1.0

---

## 使用规则

1. **何时记录：** 当某个已确认的文档/模块被发现缺少标记、标注不一致、或需要补充内容时
2. **何时修复：** 在相关 Phase 启动前、或安排专门的"文档对齐"迭代时集中处理
3. **状态管理：** `待修复` → `已修复`（附修复日期和 commit）
4. **优先级：** P0=阻塞后续开发 / P1=影响文档一致性 / P2=改善可维护性

---
